**Anik Bakshi and Arush Bodla**: *RegiMon: Monitoring the Windows Registry in Real-Time*. The Windows Registry is a hierarchical database containing key-value pairs governing core operating system behavior including security configurations, user permissions, and software privileges. Because of its central role in system operation, the Registry is a prime target for malware seeking to establish persistence. The registry has both a stable and volatile copy, and while existing tools such as RegRipper or RegShot analyze the on-disk registry through point-in-time snapshots, there is a significant gap in monitoring the volatile registry which allows malicious activity to go unnoticed.This project presents RegiMon, a lightweight real-time registry monitoring application built in Rust using the Tauri framework. RegiMon takes advantage of the Windows API call RegNotifyChangeKeyValue() to continuously monitor the volatile RAM copy of the registry, detecting modifications, deletions, and insertions to the registry as they occur. Upon detecting a change, the application alerts the user and offers immediate remediation options, including the ability to undo suspicious modifications. RegiMon runs as a system tray application, minimizing performance overhead and avoiding
interference with existing security software.Testing was conducted in a sandboxed Windows 10 environment using manual registry edits and system settings changes. RegiMon successfully detected all changes in real time, demonstrating its viability as an early detection mechanism. Following these tests, the Love Bug virus was used against RegiMon, and RegiMon was able to prevent the persistence mechanisms Love Bug
attempted to create. Future work will extend monitoring to a larger set of keys and implement persistent logging.

**Brian Tay**: *Nutsolver: Computing Optimal Poker Strategies*. This project presents the design and implementation of a solver that calculates the game theory optimal strategy in No-Limit Texas Hold’em Poker. Unlike perfect information games like chess where everything about the current game state is known, the two cards of the opponent are hidden in poker. However, a Nash equilibrium spot is known to exist when there are two players, meaning that there is an optimal, unexploitable strategy of playing. The goal of this project is to develop an open-source solver that can accurately and
efficiently calculate said strategy.
Poker can be represented as a game tree, where each node contains information on the current gamestate, and a player’s strategy in a given spot. In each leaf node, a player wins the pot; from there, the expected value of each previous node can be calculated by backtracking. Chance-sampling Monte Carlo Counterfactual Regret Minimization (CFR) is used to calculate Nash equilibrium by running two agents against each other that aim to exploit each other’s strategy until equilibrium is reached.
Currently, the solver is able to solve any spot in poker starting with five community cards. Given the stack size, pot size, board cards, player ranges, and bet/raise sizes, the solver is able to calculate the optimal Nash equilibrium strategy. Future enhancements include upgrading the solver so that it’s able to solve starting from earlier betting rounds, implementing other Monte Carlo CFR algorithms, and adding features like nodelocking to be able to better analyze exploitative strategies.
